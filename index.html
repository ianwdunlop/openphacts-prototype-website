The Open PHACTS Discovery Platform has been developed to reduce barriers to drug discovery in industry, academia and for small businesses.

  • It contains all the data sources you already use, integrated and linked together so that you can easily see the relationships between compounds, targets, pathways, diseases and tissues.

  • The platform has been used to answer difficult questions in drug discovery and results have been published in peer reviewed scientific journals.

  • The platform was built in collaboration with major academic and commercial organizations involved in drug discovery such as Astra Zeneca, The Swiss Institute of Bioinformatics, The European Bioinformatics Institute, University of Vienna, VU University of Amsterdam and The University of Manchester.

  • The platform is founded on semantic web and linked data principles and uses industrial strength tools such as Virtuoso to provide fast and robust access to the chemistry and biological data sources that you trust.

  • You can access the data via the Open PHACTS API or explore it using the Open PHACTS Explorer and many other apps developed using the API.

  • The data within the platform is available in a variety of formats to suit the applications you already use. Formats include JSON, XML, TSV and RDF.

What data?

Data sources include Drug Bank, ChEBI, ChEMBL, ChemSpider, Uniprot, WikiPathways, DisGeNET, neXtProt and Gene Ontology.

The Open PHACTS Discovery Platform was built for science. Using all the data sets that you already use but integrating and linking them together allows you to build better analysis pipelines.

What do you mean by linking?

Every data provider has a different identifier for the same thing and each entity (compound, target pathway etc) in the plaform has one or more of these identifiers or URIs. For example, for Sorafenib the DrugBank identifier is http://www.drugbank.ca/drugs/DB00398, the ChemSpider identifier is http://www.chemspider.com/187440 and the Bio2RDF one is http://bio2rdf.org/drugbank:DB00398. In fact Sorafenib has over 30 of these URIs. Each data provider also provides a little bit of the puzzle about what properties an entity has. By linking these identifers together the Open PHACTS Discovery Platform gives you the complete data picture and not that which is specific to one provider. All the provenance about where each property comes from is provided so that you can always go back to the source. For example, for a compound the data can come from DrugBank, ChEMBL, Chemspider and ConceptWiki.

Many drug discovery use cases were used to drive the development of the platform and these are now available as workflows which can be re-used and repurposed to answer your scientifc questions. Examples include:

  • For a given compound, give me the interaction profile with targets

  • Give me all oxidoreductase inhibitors active <100 nM in human and mouse

  • For a target, give me all active compounds with the relevant assay data. For this target which active compounds have been reported in the literature?

The Open PHACTS data can be browsed just like any web page. After creating your application key and id you can use the interface to find compounds, targets, pathways and more. The API pages show you the actual request that you made and these can be can then be opened in a browser just like any web page. Many applications have already been developed using the Open PHACTS Discovery Platform's API. The API allows the developer to use any programming language and standard HTTP calls to fetch information about any of the entities within the platform. Libraries are already available in various languages to give developers a helping hand when developing applications.

Just like the web, the API uses HTTP. Each API request uses HTTP Get with the path to the response required and a set of parameters. Each API call requires at least your API key and application identifier with most also requiring the URI of the entity of interest.

In the example below the developer is calling the 1.4 API to return information about the compound represented by the URI http://www.conceptwiki.org/concept/38932552-111f-4a4e-a46a-4ed1d7bdf9d5. The response is requested to be in JSON denoted by the _format=json parameter. The request also contains the developer's app_key and app_id.

Example API call to return Compound Info

https://beta.openphacts.org/1.4/compound?uri=http%3A%2F%2Fwww.conceptwiki.org%2Fconcept%2F38932552-111f-4a4e-a46a-4ed1d7bdf9d5&app_id=1c22cbe7&app_key=167a3a3d8539b5d85280e7178f4e62ab&_format=json

  • OPS.js is a JavaScript library containing methods to call the Open PHACTS API and parse the responses into simple JSON objects. The library has been used extensively in the Explorer and example uses can be found in the documentation as well as in the Explorer source code.

  • The ops_gems library provides API access via Ruby and is used in the ChemBio Navigator application.

The API is documented using Swagger and there are many third party libraries for automatically generating code to access the API.

The source code for many applications can be found in GitHub. The source code underlying the platform and the API is also freely available.

The Open PHACTS Foundation is a not-for-profit membership organisation, supporting the Open PHACTS Discovery Platform: A sustainable, open, vibrant and interoperable information infrastructure for applied life science research and development.

The Open PHACTS API will always be free

The Open PHACTS API is freely available for individuals and companies (up to 5 developer accounts), as described in the Open PHACTS Foundation’s core business values. Become a member if you wish to help direct the development and strategy of the Open PHACTS Foundation. For heavy users of the API, and for those who require enhanced support / security Premium API subscription packages are available.
